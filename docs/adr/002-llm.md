# ADR-002: Groq Cloud (llama-3.3-70b-versatile) over Anthropic/OpenAI
Status: Accepted
Date: 2026-06-24

## Context
- The project is MIT open-source with zero operating budget at launch.
- Anthropic and OpenAI require paid subscriptions for production use; their free tiers are very limited.
- Groq Cloud offers a generous free tier (no credit card required) with fast inference via their LPU chips.
- Groq exposes an OpenAI-compatible API, so the client code uses the same message format.
- The model `llama-3.3-70b-versatile` is available on Groq's free tier and provides strong reasoning for tutoring tasks.

## Decision
- Primary LLM: Groq Cloud with `llama-3.3-70b-versatile`
- All LLM calls go through `packages/agents/agents/base/llm.py` — a thin wrapper that handles retries, stub fallback, and streaming
- When `GROQ_API_KEY` is not set, the system returns a helpful stub response so the demo works without secrets
- No Anthropic or OpenAI keys are used

## Consequences
- Positive: Zero-cost LLM inference for early-stage development and demo.
- Positive: Fast inference latency from Groq's LPU hardware (~200ms/token).
- Trade-off: llama-3.3-70b-versatile is not Claude or GPT-4; there may be quality gaps for complex multi-step reasoning or very nuanced tutoring.
- Trade-off: Groq free tier has rate limits (30 req/min, 14,400 req/day as of 2026). Will need to upgrade for production scale.
- Future: The `llm.py` abstraction allows swapping providers without changing agent code. Can migrate to Anthropic/OpenAI once budget allows.
