# 01 — Frontend

## Goal
Stand up the Next.js 15 (App Router, TypeScript) web app for **A Step Forward**: a learner workspace with chat against the agent backend, lesson view, dashboards, Memory Inspector, and educator/admin views. Beautiful, accessible, fast.

## In-scope files
- `apps/web/**`
- `packages/ui/**`
- `packages/schemas/**` (consume only — types are generated by 02-backend)
- `apps/web/middleware.ts` (CSP, auth)

## Out-of-scope
- Backend routes (see 02-backend-api).
- Agent logic / prompts (see 03-agents).
- DB schemas (see 04-memory / 05-graphrag / 02-backend).

## Stack to use (must)
- Next.js 15 + React 19 + **TypeScript strict**.
- Tailwind v4 + **shadcn/ui** (use the `shadcn` MCP to add components).
- Vercel AI SDK for streaming chat.
- TanStack Query for non-stream reads.
- Zustand for client state.
- Zod for boundary validation; schemas shared via `packages/schemas/`.
- Clerk for auth.
- Framer Motion for animation.
- Vitest + Playwright for tests.

## Required reading
1. `PLAN.md` §3.1, §6, §8, §11.
2. `ARCHITECTURE.md` §1–2.
3. `.cursor/rules/10-typescript-style.mdc`, `50-security.mdc`, `60-testing.mdc`.
4. `skills/add-a-frontend-page/SKILL.md`.

## Pages to build (Phase 1)
- `/` marketing landing.
- `/sign-in`, `/sign-up` (Clerk).
- `/app` learner dashboard (recent lessons, mastery summary, agents).
- `/app/chat/[agent]` agent chat with streaming.
- `/app/lessons/[lessonId]` lesson view.
- `/app/memory` Memory Inspector (list, search, edit, delete with audit).
- `/app/progress` progress dashboards (Tremor/Recharts).
- `/educator` educator dashboard.
- `/admin` admin (RBAC-gated).

## Acceptance criteria
- `pnpm install && pnpm dev` runs cleanly.
- Lighthouse perf ≥ 90, a11y ≥ 95.
- All boundaries validated with Zod.
- E2E test covers: sign-in → open chat → send message → stream response.
- All routes use Server Components by default; client islands explicit.
- CSP locked down (no `unsafe-inline` for scripts).
- Dark mode + reduced-motion + i18n scaffolding (en, he as starter).
